# Full API Testing Guide (Phases 1-6)

This walkthrough provides cURL commands to test the complete flow of the marketplace platform built up to **Phase 6**. 

*(Note: Phases 1 and 2 were project scaffolding and database design, so they contain no APIs. API implementation started in Phase 3.)*

> **Disclaimer**: In the commands below, remember to replace placeholders like `<token>`, `<code>`, `<buyer-token>`, `<seller-token>`, `<admin-token>`, `<listing-id>`, and `<offer-id>` with the actual values generated by your server. Auth endpoints will return session cookies/access tokens that you'll use in subsequent `Bearer` headers.

---

## Part 1: Authentication API (Phase 3)

### 1. User Signup
```bash
curl -X POST http://localhost:8000/auth/signup \
  -H "Content-Type: application/json" \
  -d '{
    "email": "seller@example.com",
    "password": "SecurePassword123!"
  }'
```
*(Check your console or database for the generated OTP code)*

### 2. Verify Email OTP
```bash
curl -X POST http://localhost:8000/auth/verify-email \
  -H "Content-Type: application/json" \
  -d '{
    "email": "seller@example.com",
    "code": "123456"
  }'
```

### 3. User Login
```bash
curl -X POST http://localhost:8000/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "seller@example.com",
    "password": "SecurePassword123!"
  }'
```
> Extracts the `accessToken` returned from this step to use as the `<token>` below.

### 4. Logout
```bash
curl -X POST http://localhost:8000/auth/logout \
  -H "Authorization: Bearer <token>"
```

## Part 2: KYC Engine (Phase 4)

*You must submit KYC before creating or unlocking a listing.*

### 1. Submit KYC
```bash
curl -X POST http://localhost:8000/kyc/submit \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "panNumber": "ABCDE1234F",
    "aadhaarMasked": "XXXX-XXXX-1234"
  }'
```

### 2. Get KYC Status
```bash
curl -X GET http://localhost:8000/kyc/status \
  -H "Authorization: Bearer <token>"
```

### 3. Admin: Review KYC
*(You will need to login with an Admin account or alter a user's role to ADMIN in the DB to execute this)*
```bash
curl -X PATCH http://localhost:8000/kyc/<kyc-record-id>/review \
  -H "Authorization: Bearer <admin-token>" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "APPROVED"
  }'
```

## Part 3: Listing Engine (Phase 5)

*Seller must have APPROVED KYC.*

### 1. Create a Draft Listing
```bash
curl -X POST http://localhost:8000/seller/listings \
  -H "Authorization: Bearer <seller-token>" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Awesome SaaS App",
    "assetType": "SAAS",
    "industry": "Software",
    "askingPrice": 150000,
    "revenueMonthly": 5000,
    "profitMonthly": 4500,
    "visibility": "PUBLIC",
    "ndaRequired": true
  }'
```

### 2. Submit Listing For Review
```bash
curl -X POST http://localhost:8000/seller/listings/<listing-id>/submit \
  -H "Authorization: Bearer <seller-token>"
```

### 3. Admin: Review and Approve Listing
```bash
# Valid statuses: APPROVED, CHANGES_REQUESTED, LIVE, LOCKED, SOLD
curl -X PATCH http://localhost:8000/admin/listings/<listing-id>/review \
  -H "Authorization: Bearer <admin-token>" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "APPROVED"
  }'
```

## Part 4: Buyer Interactions & Seller Publish (Phase 6)

*Buyer must ALSO sign up, log in, and get KYC APPROVED before making offers.*

### 1. Seller Publishes the Listing
*Once the admin gives it the "APPROVED" status, the seller pushes it "LIVE".*
```bash
curl -X POST http://localhost:8000/seller/listings/<listing-id>/publish \
  -H "Authorization: Bearer <seller-token>"
```

### 2. View Public Listings (No Auth Required)
```bash
curl -X GET http://localhost:8000/listings
```

### 3. Buyer: Sign NDA to Unlock (If Required)
```bash
curl -X POST http://localhost:8000/buyer/listings/<listing-id>/nda \
  -H "Authorization: Bearer <buyer-token>"
```

### 4. Buyer: Unlock Listing Details
```bash
curl -X POST http://localhost:8000/buyer/listings/<listing-id>/unlock \
  -H "Authorization: Bearer <buyer-token>"
```

### 5. Buyer: Make an Offer
```bash
curl -X POST http://localhost:8000/buyer/listings/<listing-id>/offer \
  -H "Authorization: Bearer <buyer-token>" \
  -H "Content-Type: application/json" \
  -d '{
    "price": 140000,
    "terms": "Subject to code review.",
    "expiresInDays": 7
  }'
```

### 6. Seller: Respond to the Offer
*Accepting automatically initiates a Deal Room behind the scenes as required by Phase 6.*
```bash
curl -X PATCH http://localhost:8000/seller/offers/<offer-id>/respond \
  -H "Authorization: Bearer <seller-token>" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "ACCEPTED"
  }'
```
